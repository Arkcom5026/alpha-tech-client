Current Template-only state copy:


Patch-001 certified the QuickStockPage safety gate. Template-only selections can no longer scan barcode or commit existing intake until there is a confirmed Operational Product. Therefore the phrase "เมื่อ Commit ครั้งแรก" is now misleading because Commit is intentionally unavailable for Template-only products.

## 2. Recommended replacement wording

Recommended minimal replacement for the white/amber explanation box only:

```txt
สถานะปัจจุบัน:
สินค้านี้ยังเป็น Template เท่านั้น และยังไม่ใช่ Operational Product ของร้าน

ก่อนรับเข้า Stock ต้องมีขั้นตอนถัดไป:
✓ สร้างหรือ Clone Product เข้า Branch ปัจจุบัน
✓ สร้าง/บันทึก BranchPrice ของร้าน
✓ จากนั้นจึงรับสินค้าเข้า Stock Runtime ได้
```

Optional shorter version:

```txt
สินค้านี้ยังเป็น Template เท่านั้น
ต้องสร้าง Operational Product และ BranchPrice ของร้านก่อน จึงจะรับสินค้าเข้า Stock ได้
```

Recommended choice:

Use the first version because it preserves the user's mental model without promising that the current Commit action performs clone/create.

---

## 3. User-facing rationale

The user needs to understand three facts immediately after selecting a Template-only product:

1. The selected item is not yet a branch product.
2. Existing stock intake is blocked by design.
3. A future create/clone step is required before receiving stock.

The replacement wording keeps the NOT CREATED state clear and aligns with certified behavior:

```txt
Template-only product cannot be received until an Operational Product exists for the branch.
```

It also avoids implying that the existing Commit button creates product, creates BranchPrice, and receives stock in one action.
